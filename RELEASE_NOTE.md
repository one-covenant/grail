# v0.0.60

This release fixes a validator bug where honest miners were being penalized by the copycat detector when another miner copied their rollouts.

- **Copycat detection no longer punishes victims.** Previously, when two miners shared rollouts above the overlap threshold, both sides were zeroed. Now the validator uses S3 upload timestamps to identify which miner uploaded later (the copier) and gates only that side. If direction cannot be determined, neither side is gated. Validators upgrading to this version will immediately stop zeroing honest miners who are being scraped by sybil operators.
- **Cross-window attribution persists across the submission interval.** The detector now archives per-miner upload times so that interval-scope violations can still be directionally attributed even when one miner is absent from the current window. Previously, an absent victim caused the detector to lose direction and abstain for the entire accumulated interval.
- **No action required from miners.** This is a validator-only change with no protocol version bump.
- Docs updated to focus on single-GPU environments for miner and validator setup.

## v0.0.59

This release ships a fully migrated pipelined SGLang reference miner, raises the per-window unique-rollouts cap from 1,500 to 5,000, and ships the FSDP2 + DDP + DILOCO + PULSE-DiLoCo distributed trainer alongside a deep round of proof and validator hardening.

- **Reference miner fully migrated to pipelined SGLang.** The legacy single-GPU `AgentEnvLoop` path is removed from the codebase, the default backend flips `vllm` → `sglang`, and `GRAIL_PIPELINE_ENABLED` is deleted. Miners are free to fork the repo and run any architecture they want, this is just the reference build the team maintains.
- **Reference build now requires at least two GPUs** (one for SGLang generation, one for proof recompute). Set `GRAIL_PIPELINE_PROOF_GPU_ID` and `GRAIL_PIPELINE_VLLM_GPU_ID` to physical indices in `.env`. Validated at startup; the miner exits on out-of-range or pipeline-init failure instead of silently falling back.
- **New miner Dockerfile** (`docker/Dockerfile.miner`) builds on the validator base via `ARG GRAIL_BASE_IMAGE=ghcr.io/one-covenant/grail:latest` and force-reinstalls grail's pinned `transformers` / `tokenizers` / `torch` over SGLang 0.5.10's metadata pins so SGLang's compiled bits stay loadable. **Built locally on your GPU host**, not published to ghcr (SGLang's full closure blows the GitHub Actions runner disk budget). See `docs/miner.md` for the build command.
- **Per-window cap raised 1,500 → 5,000 unique rollouts** (period cap 60,000 over the 12-window submission interval). The new ceiling matches what the SGLang reference miner can actually produce.
- **`GRAIL_BURN_PERCENTAGE` raised 80% → 90%.** The non-burn share that gets split among miners shrinks from 20% to 10%, so a miner at 100% of cap now takes home **half** of what they earned before in absolute terms. Anything below 100% of cap leaks even more to burn than under the old curve. This is the bigger of the two economics changes.
- **Mining throughput now sustains 4,477 unique rollouts per 360s window** (12.44 r/s steady state, 97% of the pure SGLang TP=4 generation ceiling, ~35× the previous ~128 r/window main-branch baseline). E2e measured on 8x A100 SXM4 with Qwen2.5-7B.
- **SGLang backend rewritten to use the native `/generate` endpoint** with `input_ids` directly via `httpx`. No BPE re-tokenization, no stop-token stripping, exact `output_ids` returned. A binary `_cap_max_tokens` clamp on overlong prompts means the validator's `termination_valid` check no longer fails on partially-clamped completions.
- **Proof loop runs `log_softmax + gather` on the proof GPU.** Per-sequence PCIe traffic drops from ~1.2 GB to ~16 KB (5× per proof GPU), the change that unblocked multi-worker scaling.
- **Persistent HF proof subprocess workers across windows** (`grail/mining/proof_worker.py`). Numpy-packed `ProofJob` (sequences as `int32`) gives ~50× faster pickling than `list[list[int]]`. The next iteration's gen overlaps the previous iteration's proof submit via `asyncio.to_thread`.
- **Checkpoint weights hash drops SHA-256 + `tobytes()` for `xxh3-128`** over a zero-copy `memoryview` (~10× faster on a 7B model). Stamped on every FULL publish so consumers can verify integrity post-download.
- **Trainer-published sampling policy is now a hard contract.** `CheckpointMetadata.generation_params` MUST contain `max_tokens`, `temperature`, `top_p`, `top_k`, and `repetition_penalty`. Trainer's per-checkpoint default `max_tokens` lowered 8192 → 2048 to match production policy.
- **`CheckpointMetadata.thinking_mode` is now a required protocol field** (`"native"` or `"instructed"`). The validator overrides the process-wide `GRAIL_THINKING_MODE` from the resolved checkpoint, so miner and validator can no longer drift on host-env state.
- **New `GenerationParams.from_checkpoint_metadata` trust boundary** parses the trainer-published dict and raises `ProtocolViolationError` (a `RuntimeError` subclass) on missing or malformed fields. Replaces bare `assert` invariants on the data path so failures stay loud under `python -O`.
- **Validator resolves the per-window env config ONCE per window** from the trainer's checkpoint and skips the entire window if any field is missing or malformed. The previous per-miner fallback silently scored every miner against an unverifiable baseline.
- **`MAX_NEW_TOKENS` renamed to `MAX_NEW_TOKENS_PROTOCOL_CAP=8192`** to make explicit that this is the immutable network bound. The trainer's per-checkpoint `max_tokens` is the actual operating limit.
- **Latent seed-derivation bug fixed.** When a miner dropped a rollout group, the validator's reconstructed seeds drifted by one slot from the miner's, hard-failing every prompt-validation check on that miner. The validator now uses the miner-published `rollout_group` integer directly.
- **Distributed training is first-class.** New `grail/trainer/distributed/` module with four selectable strategies (FSDP2 default, DDP, DILOCO, PULSE-DiLoCo v6) dispatched via `GRAIL_DIST_STRATEGY` / `GRAIL_DIST_NPROC`.
- **FSDP2 measured 2.18× speedup** on 2x B200 / Qwen3-8B (192s vs 419s per epoch, 31.9K vs 14.6K tok/s, **34.79% MFU vs 15.97%**, 109 GB vs 182 GB peak).
- **PULSE-DiLoCo v6 fix** transmits raw FP32 values at fired indices (BF16 acts as a sparsity mask only, removes quantization overshoot) and calls `malloc_trim(0)` after each sync to reclaim 10-50 GB of glibc heap.
- **SGLang inference server now hot-reloads checkpoints** between training iterations, no full restart per checkpoint.
- **Snapshot gating only updates `snapshots/latest/` after a DILOCO outer sync** so the network never sees non-consensus weights.
- **GRAIL proof v5.** Top-K indices sorted by **position** (not magnitude rank), making the sketch invariant to GEMM tile reordering across batch sizes. `PROOF_TOPK` 32→16, `PROOF_SKETCH_TOLERANCE_BASE` 30→6000, growth 3.0→5.0. Validated across 6 models / 3 GPU types with 545M+ proof positions, 27% headroom on the worst-case passing pair, ~10⁻¹⁶⁷ forgery probability at K=32. **Validators only accept v4 and v5; v1/v2 are rejected, coordinate the network upgrade.**
- **FA2 is now pinned for both miner and validator** (immutable in `grail/protocol/constants.py`, no env override on the proof path). The Dockerfile preinstalls flash-attn 2.8.3 from the prebuilt Dao-AILab wheel (covers A100, H100, B200, no source compile).
- **Validator refuses silent CPU fallback.** `get_model()` no longer pins 60 vCPUs and exhausts 150 GB RAM (a real production incident on an 8B FP32 model). The CLI runs a startup GPU probe before any subtensor or R2 work.
- **LogprobValidator switched to a single-knob median importance-sampling check** (`median(exp(|Δlp|) − 1) ≤ 0.10` over K=32 positions). Calibrated on **430,650 honest trials** across A100/B200/L40 with **0 false positives**, max honest median dev 0.066 (50% headroom).
- **DistributionValidator now replays HF's `RepetitionPenaltyLogitsProcessor` + `TemperatureLogitsWarper`** on cached logits before reading chosen-token probs. Fixes systematic `min_prob=0.0` failures on rollouts that used non-default sampling params.
- **SGLang `extra_body` passthrough restored** after a regression that fell back to model defaults (Qwen3 `top_k=20`).
- **SGLang piecewise CUDA graph capture disabled on Blackwell** to work around `cudaErrorIllegalAddress` during warmup.
- **Validator compose hardened.** `docker-compose.validator.yml` adds `NVIDIA_VISIBLE_DEVICES`, `NVIDIA_DRIVER_CAPABILITIES`, `shm_size: 2g`, pins device IDs explicitly via `GRAIL_VALIDATOR_GPU_IDS`, supports `GRAIL_VALIDATOR_IMAGE` for tag/sha pinning, runs with `pid: host`, and uses `uv run --no-sync grail`. **Validator operators must redeploy with the updated compose file.**
- **Trainer chunked-logits path takes the logit/log-prob tensors from ~10 GB to ~312 MB peak** by applying `lm_head` in sequence-dimension chunks (default-on via `GRAIL_TRAINER_CHUNKED_LOGITS=1`).
- **Pluggable advantage estimators**: `grpo`, `dr_grpo`, `dapo`. Plus token-budget micro-batching via greedy LPT bin-packing (under 0.1% imbalance), opt-in FA4 attention for Blackwell, opt-in Liger kernels (RMSNorm/RoPE/SwiGLU), and `torch.compile` on the base transformer with fixed-shape padding.
- **`grail/shared/constants.py` deleted and split** into `grail/protocol/constants.py` (immutable, **zero `os.getenv` calls**) and `grail/shared/config.py` (deployment-tunable). Enforces the protocol-constants-are-immutable rule across the ~80 import sites that moved.
- **Test infrastructure**: pytest-xdist parallel CI with a separate serial job for tests touching global singletons, filelocks around HF model downloads, session-scope Qwen2.5-1.5B fixture with per-test `deepcopy` (no more per-test 1.5B reload), 75 new distributed-strategy tests passing on CPU without NCCL.
- **Required**: Python 3.12 or newer (3.10/3.11 dropped, `uv.lock` regenerated, backport packages removed). Pre-built docker images cover this; build-from-source operators must update their venv. Importers must move from `grail.shared.constants` to `grail.protocol.constants` or `grail.shared.config`. Validator metric scrapers reading `logprob_mismatches` must switch to `logprob_median_dev` and `logprob_max_dev`.
